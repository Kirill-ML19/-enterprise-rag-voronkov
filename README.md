# RAG CHALLANGE

Этот проект реализует **Retrieval-Augmented Generation (RAG)** систему для извлечения структурированной информации из годовых финансовых отчетов в формате PDF.

Система находит релевантные фрагменты документов, выполняет их перерангирование и использует языковую модель для генерации ответа с указанием ссылок на страницы исходных документов.

---

## Обзор проекта

Пайплайн системы состоит из следующих этапов:

1. **Обработка документов**
   - PDF документы парсятся и разбиваются на текстовые фрагменты (chunks).

2. **Создание эмбеддингов и индексация**
   - Текстовые фрагменты преобразуются в векторные представления (embeddings).
   - Векторы сохраняются в векторную базу данных.

3. **Поиск (Retrieval)**
   - Для каждого вопроса система извлекает наиболее релевантные фрагменты текста.

4. **Переранжирование (Reranking)**
   - Найденные фрагменты сортируются по степени релевантности.

5. **Генерация ответа**
   - Языковая модель генерирует итоговый ответ на основе выбранного контекста.

6. **Формирование submission**
   - Ответы форматируются и проходят валидацию перед созданием итогового файла.

## Установка

В этом разделе описано, как установить все необходимые зависимости и подготовить окружение для запуска проекта.

---

# 1. Требования

Перед установкой убедитесь, что на вашем компьютере установлены:

- Python **3.10 или новее**
- **pip**
- **Docker** (для запуска векторной базы данных)
- **Git**

---

# 2. Клонирование репозитория

Сначала скачайте проект:

```bash
git clone https://github.com/Kirill-ML19/-enterprise-rag-voronkov
cd project_name
```

---

# 3. Создание виртуального окружения

Рекомендуется использовать **виртуальное окружение**, чтобы зависимости проекта не конфликтовали с системными пакетами.

Создание окружения:

```bash
python -m venv venv
```
Активация виртуального окружения

```bash
source venv/bin/activate
```

---

# 4. Установка Python зависимостей

Теперь установите все необходимые библиотеки:

```bash
pip install -r requirements.txt
```

---

# 6. Запуск векторной базы данных

Проект использует **Qdrant** как векторную базу данных.

```bash
docker run -p 6333:6333 qdrant/qdrant
```

После запуска база будет доступна по адресу:

```
http://localhost:6333
```

---

# 7. Настройка переменных окружения

Создайте файл `.env` в корне проекта.

```bash
touch .env
```

Добавьте в него API ключ:

```
OPENROUTER_API_KEY=your_api_key_here
```

Этот ключ используется для доступа к языковой модели.

---

# 8. Запуск

Парсинг документов

```bash
python -m scripts.parse_pdfs
```

Формирование векторной базы данных

```bash
python -m scripts.build_index
```

Запуск RAG

```bash
python src/main.py
```
